# Flux VPN Merger — Автоматический Объединитель VPN-Подписок

Этот проект представляет собой готовое, бесплатное решение для автоматического слияния, очистки и фильтрации VPN-конфигов из более чем 100 различных публичных подписок (включая протоколы VLESS, VMess, Reality, Trojan и Shadowsocks). 

Система автоматически удаляет дубликаты серверов по IP и портам, разделяет их по протоколам, вычленяет узлы, оптимизированные для обхода блокировок в РФ (DPI, SNI, ТСПУ), и предоставляет удобный веб-интерфейс для генерации ссылок.

Решение разворачивается **бесплатно** на платформе GitHub (GitHub Actions + GitHub Pages).

---

## 🚀 Как запустить проект у себя на GitHub (Пошагово)

1. **Создайте репозиторий**:
   - Войдите в свой аккаунт на GitHub и создайте новый публичный репозиторий с любым именем (например, `flux-vpn-merger`).

2. **Загрузите файлы**:
   - Загрузите все файлы из этой папки в ваш новый репозиторий. Структура должна быть следующей:
     ```
     ├── .github/workflows/update.yml  # Файл автообновления
     ├── merge.py                     # Скрипт слияния
     ├── index.html                   # Веб-страница
     ├── styles.css                   # Стили веб-страницы
     ├── app.js                       # Скрипт веб-страницы
     └── README.md                    # Инструкция
     ```

3. **Настройте разрешения GitHub Actions** (Критически важно!):
   - В вашем репозитории перейдите в **Settings** (Настройки) → **Actions** → **General**.
   - Прокрутите страницу в самый низ до раздела **Workflow permissions**.
   - Переключите маркер на **Read and write permissions** (Разрешить чтение и запись).
   - Нажмите **Save**. Это позволит автообновлению записывать готовые подписки в репозиторий.

4. **Включите GitHub Pages** (Для работы сайта):
   - Перейдите в **Settings** → **Pages** (в левом меню).
   - В разделе **Build and deployment** выберите Source: **Deploy from a branch**.
   - В выпадающем меню Branch выберите **`main`** и папку **`/ (root)`**.
   - Нажмите **Save**. Через 1-2 минуты ваш сайт станет доступен по ссылке `https://<ваш-логин>.github.io/<имя-репозитория>/`.

5. **Первый запуск произойдет автоматически**:
   - Поскольку запуск настроен на событие пуша (`on: push`), скрипт слияния выполнится сам сразу после того, как вы загрузите файлы в репозиторий.
   - Вы можете следить за процессом во вкладке **Actions** в верхнем меню.
   - После завершения работы (обычно занимает около 1 минуты) в вашем репозитории автоматически появится папка `dist/` со всеми готовыми файлами подписок и файлом статистики `stats.json`.

## 🛠 Как устроен скрипт слияния (`merge.py`)

Скрипт написан на чистом Python 3 и **не требует внешних библиотек** (использует стандартный `urllib`), что ускоряет его запуск и снижает вероятность сбоев на серверах GitHub Actions.

Скрипт выполняет следующие задачи:
1. Загружает базы серверов из 100+ источников в параллельном многопоточном режиме (`ThreadPoolExecutor`).
2. Автоматически определяет тип кодирования (обычный текст или Base64) и расшифровывает его.
3. Очищает список от дубликатов по ключу `(протокол, хост, порт)`.
4. Сортирует ноды по отдельным файлам:
   - `dist/all_base64.txt` — все сервера (Base64).
   - `dist/vless_base64.txt` — только VLESS / Reality.
   - `dist/vmess_base64.txt` — только VMess.
   - `dist/trojan_base64.txt` — только Trojan.
   - `dist/ss_base64.txt` — только Shadowsocks.
   - `dist/russia_base64.txt` — только сервера, специально оптимизированные для обхода ТСПУ/DPI в РФ (из российских источников или с соответствующими тегами).
5. Создает файл `dist/stats.json` со статистикой для отображения на сайте.

## 🔄 Расписание обновлений

Скрипт автоматически запускается на серверах GitHub каждые **6 часов** благодаря настройке cron в файле `.github/workflows/update.yml`. Также вы можете запустить его вручную в любой момент во вкладке Actions или просто сделав новый коммит.
